@Log4j2 , 애너테이션 이용해서,
로깅 기록을 할 예정.
출력 대상, 콘솔, 파일, 디비에 출력이 가능함.

Apache Log4j2 , 외부 라이브러리,
롬복에서, 라이브러리 이용해서 호출을 할 예정.
로그를 기록 할 때, 중요도,
1)개발 할 때 vs 2)운영할 때,
로그를 기록하는 방법이 다름.

예시)
1) 개발 할 때, 구체적으로 정보를 요구를 많이 하는 편.
2) 운영 할 때,  뭔가 시스템에 장애가 발생하는 경우에만 기록을 할려고 함.
만약, 운영 단계 로깅 기록을 debug로 세세하게 기록을 한다.?
서버가 자원을 많이 소모를 해서, 제 기능을 못함.

중요도의 관계, 우선순위에 따라서, 기록하기.
로그레벨 중요도에 따라서, 기록을 함.
가장 우선 순위가 높은 순서,
1. FATAl, 치명적인 오류,
2. ERROR, 일반적인 오류, //
3. WARNING, 경고 단계 // 운영시 사용.
4. INFO, 일반적인 정보 // 개발시 사용
5. DEBUG, 상세하게 출력하는 단계
6. TRACE, 호출하는 순서까지 구체적으로 기록하는 단계
-> 중요도에서, 해당 레벨의 상위 까지만 출력을 함.
예시) INFO 설정하면, 1,2,3 번 다 포함.
예시2) ERROR 설정하면,ㅣ 1,2 번 포함.
